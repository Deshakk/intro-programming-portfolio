## My understanding of how C# gets executed behind the scenes
C# has a compiler called Roslyn, and it translates all the code into something called Intermediate Language (IL). Then the IL code is stored in an exe or DLL file. When the program starts running, the CLR (Common Language Runtime) takes over. The CLR is a virtual machine. And then lastly, inside the CLR, there's a JIT (Just-In-Time) compiler that translates the IL code into binary so that the computer's CPU can read it.
